# Talk Packaged Local ASR Service Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Talk's `streaming_service` path shippable by including the local ASR daemon in the Windows desktop release package.

**Architecture:** Keep the desktop-to-ASR boundary as the existing loopback WebSocket protocol. The release publisher builds `talk-local-asr-sherpa.exe`, places it in `.internal`, records it in the manifest/support files/checksums, and emits a desktop config that points at `streaming_service` so a release package contains both the GUI and the local ASR service binary.

**Tech Stack:** PowerShell release publisher, Rust workspace binaries, TOML config generated by `Publish-TalkRelease.ps1`, existing manifest validator.

---

## File structure

- Modify `Talk/scripts/Publish-TalkRelease.ps1`: build/package `talk-local-asr-sherpa.exe`, record it as an internal support binary, and generate desktop config with `[speculative.streaming_service]`.
- Modify `Talk/docs/LOCAL_STREAMING_ASR_PROTOCOL.md`: update dry-run daemon command to document `--dry-run-partial-text`.
- Validate with `Publish-TalkRelease.ps1`, `Test-TalkReleaseManifest.ps1`, and file existence checks.


- [x] **Step 4: Record daemon in release metadata**

Add a support file record:

```powershell
[pscustomobject]@{
    kind = 'local-asr-daemon'
    path = '.internal/talk-local-asr-sherpa.exe'
}
```

and include `talk-local-asr-sherpa.exe` in `BUILD_INFO.txt` artifact names.

- [x] **Step 5: Generate a local-first desktop config**

Extend `New-TalkReleaseDesktopConfigContent` so the packaged config includes:

```toml
[speculative]
enabled = true
local_asr = "streaming_service"
cloud_correction = "provider_text_processor"
max_patch_age_ms = 2000
max_auto_patch_edit_ratio = 0.25

[speculative.streaming_service]
endpoint = "ws://127.0.0.1:53171/asr"
sample_rate_hz = 16000
channels = 1
connect_timeout_ms = 1000
idle_timeout_ms = 3000
final_timeout_ms = 7000
```

This keeps the release config aligned with the local-first architecture; service lifecycle automation remains a later task.
